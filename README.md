# Development of a Library Seat Occupancy and Concentration Monitoring System Using Raspberry Pi

라즈베리파이에 인체감지 센서, 카메라, 능동 부저, LED를 연결하여 시험기간 대학교 도서관 열람실의 좌석 독점 문제와 학생들의 졸음 현상을 방지하는 통합 모니터링 시스템을 만드는 프로젝트입니다.

> 본 프로젝트는 센서를 통한 실시간 자리 확인과 카메라 영상 인식(OpenCV)을 활용하여 졸음을 감지하며,
> Flask 웹 서버를 통해 외부에서도 스마트폰으로 좌석 현황을 확인할 수 있도록 제어하는 실습을 다룹니다.

## 개요

라즈베리파이에 오픈소스 패키지와 물리 센서들을 연결하여 열람실 좌석의 착석 여부를 판별하고, 공부 중인 학생의 상태에 따라 외부 하드웨어를 제어하는 복합 장치를 만드는 것이 목적입니다. 인체감지 센서의 디지털 입력 프로그래밍을 익히고, 제공된 XML 분류기 파일들을 코드로 불러와 객체를 탐지하는 원리를 이해합니다. 또한, 전체 이미지 연산의 부담을 줄여주는 관심 영역(ROI) 추출 기법을 학습하며, Flask 웹 서버를 통해 센서와 비전 데이터가 실시간 웹 대시보드로 동기화되는 과정을 구현합니다.

## 핵심 특징

- **실시간 좌석 확인**: 인체감지 센서로 사람이 있는지 확인하여 "자리 비움", "이용 중" 상태를 판단함
- **영상 처리 속도 향상**: 얼굴 영역(ROI)만 잘라내어 눈을 찾음으로써 라즈베리파이의 연산 부담을 줄임
- **소리 및 불빛 경보**: 학생이 눈을 감거나 졸아서 감지된 눈이 1개 이하가 되면 능동 부저와 LED가 켜져서 잠을 깨워줌
- **실시간 웹 모니터링**: Flask 웹 서버와 멀티스레딩 기술을 사용하여, 카메라가 작동하는 중에도 스마트폰으로 좌석 현황과 졸음 횟수를 실시간으로 볼 수 있음

## 하드웨어 아키텍처

본 프로젝트는 [라즈베리파이의 BCM 핀 배열](https://www.raspberrypi.com/documentation/computers/raspberry-pi.html#gpio)을 기준으로 다음과 같이 설계되었습니다.
- **인체감지 센서**: GPIO 16 (VCC는 3.3V에 연결)
- **능동 부저**: GPIO 20 (VCC는 3.3V에 연결)
- **빨강 LED**: GPIO 21
- **USB 웹캠**: USB 포트에 연결

## 회로 구성 이론

- **인체감지 센서 원리**: 사람이 움직일 때 발생하는 변화를 센서가 감지하여 라즈베리파이에 디지털 신호를 보내줍니다.
- **관심 영역(ROI) 추출**: 카메라 화면 전체에서 눈을 찾으면 속도가 매우 느려집니다. 그래서 먼저 얼굴을 찾고, 그 얼굴 안에서만 눈을 찾도록 구역을 잘라내어(행렬 슬라이싱) 제한된 성능의 라즈베리파이에서도 끊김 없이 작동하게 만들었습니다.

## 실행 및 설치

### 1. 가상환경 및 필수 라이브러리 설치

라즈베리파이 터미널에서 가상환경을 켜고, 필요한 파이썬 라이브러리들을 설치합니다.

# 가상환경 켜기
source venv/bin/activate

# 필수 라이브러리 설치
pip install opencv-python flask gpiozero lgpio

### 2. 서버 실행

터미널에서 소스코드가 있는 디렉토리로 이동 후 'python3 library_monitor.py' 명령어를 입력합니다.

### 3. 종료(Interrupt)
- 실행 중 Ctrl + C를 입력하면 KeyboardInterrupt가 발생하며 프로그램이 종료됩니다.

## 주요 이슈 해결
- 인체감지 센서와 능동 부저가 둘 다 GPIO 16번으로 겹쳐서 장치가 오작동하는 문제를 확인했습니다. 부저를 비어있는 GPIO 20번 구멍으로 옮겨 꽂고 코드를 수정하여 충돌을 해결했습니다.
- 가상환경에서 실행할 때 라즈베리파이의 핀을 찾지 못하는 에러가 발생했습니다. 터미널에 하드웨어 제어를 돕는 `lgpio` 라이브러리를 추가로 설치하여 해결했습니다.
- 화면 전체를 실시간으로 계속 인식하면 제한된 성능의 라즈베리파이에서 연산이 밀려 느려지는 현상이 있었습니다. 관심 영역(ROI) 슬라이싱 기법을 적용해 얼굴 안쪽만 집중 조사하도록 만들어 속도를 크게 높였습니다.
- 프로그램 종료 메커니즘의 한계와 인터럽트(SIGINT) 이해: 임의의 키 종료를 위해 외부 라이브러리 설치를 시도했으나, 별도 설치 없이 운영체제 차원에서 즉각 반응하는 Ctrl + C 인터럽트가 하드웨어 제어 및 안전 종료에 가장 효율적임을 학습함.
